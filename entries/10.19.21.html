CodeForces Virtual Contest

Today, the UC Davis ICPC team set up a virtual contest.

Here's a breakdown of each problem, and how I approached them. Note that these are not the best, most simple, or even most correct solutions. They just happened to be ones that passed the test cases I was given.

A. Computer Game

The problem is simple enough. Given a 2 by n grid with certain blocked off units, is it possible to reach the lower rightmost square from the upper leftmost square with the only legal move being moving one unit in any direction without skipping any squares?

In this case, I immediately saw the solution. The grid is not walkable if and only if there are two adjacent blocked off units in both rows, which is a condition you can check with one pass of the array.

B. Groups

Clearly not an exercise in abstract algebra, this question demands to know whether or not we can line up the schedules of 2k students where k is an integer given which days they are free. The catch being that we need to find two distinct groups of people whose schedules match on at least one day.

I approached this problem using sets. A set is a collection of elements without any repeats. The library was rather helpful in accomplishing this.

Essentially, I defined an operation for the union of two sets, which is a new set that includes elements from both of the sets. Again, this eliminates all duplicate elements.

Why is this important?

We want to determine whether or not there exists two days such that the set of people who are free on day X and day Y contain no common elements. In other words, the union of these sets must contain exactly 2k elements.

With that, we can create an array of sets and populate it with the index of each person, removing duplicates whenever necessary.

Finally, we find all sets with size greater than or equal to k, and take their pairwise unions. As soon as we find a set with cardinality k, we are done. If we cannot find one, there is no optimal schedule that exists.

C. Delete Two Elements

Sigh. This problem really frustrated me. I thought my solution worked but it wouldn't pass the third test case.

The problem is simple. Given an array of integers, there's some real number r that corresponds to the arithmetic mean of the elements of the array. Can we remove two numbers from the array such that the arithmetic mean of the elements of the new array is still r?

This reduces to a two-sum problem. Just find all pairs in the array that add up to 2*s/n. Simple, right?
